1Ô∏è‚É£ SQL vs NoSQL
Feature	SQL (Relational DB)	NoSQL (MongoDB)
Structure	Tables with rows & columns	Collections with documents
Schema	Fixed schema	Dynamic/flexible schema
Query Language	SQL	MongoDB Query Language (JSON-like)

Key takeaway:
MongoDB is document-oriented, schema-flexible, and scales horizontally easily.

2Ô∏è‚É£ Document Structure
üîπ Documents = JSON-like objects

A single record in MongoDB is called a document.

Example:

{
    "_id": 1,
    "name": "Vanshika",
    "age": 22,
    "skills": ["Java", "Python", "MongoDB"],
    "address": { "city": "Delhi", "state": "Delhi" }
}


_id ‚Üí unique identifier

Values can be strings, numbers, arrays, objects

Flexible ‚Üí different documents in same collection can have different fields

üîπ Collections

Like tables in SQL

Stores multiple documents

db.employees.insertOne({
    "name": "Alice",
    "age": 30,
    "department": "IT"
})


db.employees.find() ‚Üí retrieve documents

3Ô∏è‚É£ BSON

Binary JSON

MongoDB stores documents in BSON format internally

Supports:

Extra data types (Date, Binary)

Faster parsing than plain JSON

Example:

{ "name": "Bob", "joined": ISODate("2026-01-27") }

4Ô∏è‚É£ Why MongoDB?
üîπ Advantages

Flexible Schema

No need to predefine table structure

Add/remove fields anytime

Horizontal Scalability

Sharding ‚Üí split large datasets across servers

Document Model = Natural Mapping

Nested data stored as documents (JSON ‚Üí easy mapping for apps)

High Performance

Reads/writes are fast for large unstructured data

Aggregation & Indexing

Powerful aggregation pipeline (like GROUP BY)

Secondary indexes for fast queries

Popular in modern apps

Node.js, Python, MEAN/MERN stack, big data

üîπ Quick MongoDB Commands
// Show collections
show collections

// Insert document
db.employees.insertOne({name: "Alice", age: 30})

// Find all
db.employees.find()

// Find with condition
db.employees.find({age: {$gt: 25}})

// Update
db.employees.updateOne({name: "Alice"}, {$set: {age: 31}})

// Delete
db.employees.deleteOne({name: "Alice"})
